# Spring 事务

## 简介

> Spring 本身并没有事务，Spring 的事务是对数据库事务的包装。

## ACID 特性

| 特性 | 说明 | 类比 |
| :--- | :--- | :--- |
| **原子性** | 要么全部成功，要么全部回滚 | 转账：扣钱+加钱必须同时成功 |
| **一致性** | 事务前后数据状态保持一致 | 总金额不变 |
| **隔离性** | 多个事务互不干扰 | 你转账不影响我查询 |
| **持久性** | 提交后数据永久保存 | 断电也不会丢失 |

## 事务管理方式

### 编程式事务（不推荐）

```java
@Service
public class UserService {
    @Autowired
    private TransactionTemplate transactionTemplate;

    public void saveUser() {
        transactionTemplate.execute(status -> {
            // 业务代码
            userDao.insert(user);
            userLogDao.insert(log);
            return null;
        });
    }
}
```

> 缺点：代码侵入性强，业务代码与事务代码耦合。

### 声明式事务（推荐）

```java
@Service
public class UserService {

    @Transactional
    public void saveUser(User user) {
        userDao.insert(user);
        userLogDao.insert(log);  // 任一失败，全部回滚
    }
}
```

> 优点：无侵入，只需添加注解。

## @Transactional 详解

| 属性 | 说明 | 默认值 |
| :--- | :--- | :--- |
| propagation | 事务传播行为 | Propagation.REQUIRED |
| isolation | 事务隔离级别 | Isolation.DEFAULT（数据库默认） |
| timeout | 超时时间（秒） | -1（不超时） |
| readOnly | 是否只读 | false |
| rollbackFor | 触发回滚的异常 | RuntimeException 和 Error |
| noRollbackFor | 不触发回滚的异常 | 无 |

```java
@Transactional(
    propagation = Propagation.REQUIRES_NEW,
    isolation = Isolation.READ_COMMITTED,
    timeout = 30,
    readOnly = false,
    rollbackFor = {BusinessException.class},
    noRollbackFor = {NoticeException.class}
)
public void transferMoney(String from, String to, BigDecimal amount) {
    // 业务逻辑
}
```

## 事务传播行为

当事务方法调用另一个事务方法时，Spring 如何处理事务。

| 传播行为 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **REQUIRED** | 当前有事务则加入，无则新建（默认） | 大多数业务方法 |
| **REQUIRES_NEW** | 始终新建事务，挂起当前事务 | 独立操作日志、审计 |
| **SUPPORTS** | 有事务则加入，无则以非事务执行 | 查询方法 |
| **NOT_SUPPORTED** | 以非事务执行，挂起当前事务 | 不需要事务的操作 |
| **MANDATORY** | 必须在事务中执行，否则抛异常 | 强制事务的方法 |
| **NEVER** | 必须在非事务中执行，否则抛异常 | 禁止事务的方法 |
| **NESTED** | 嵌套事务（Savepoint 机制） | 部分回滚的场景 |

```java
@Service
public class OrderService {

    @Transactional(propagation = Propagation.REQUIRED)
    public void createOrder() {
        orderDao.insert(order);
        paymentService.pay();  // 支付方法会加入当前事务
        logService.saveLog();  // 日志方法要求新事务
    }
}

@Service
public class PaymentService {
    @Transactional(propagation = Propagation.REQUIRED)
    public void pay() { }  // 加入调用方事务
}

@Service
public class LogService {
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void saveLog() { }  // 独立事务，即使主事务回滚，日志也会提交
}
```

## 事务隔离级别

解决并发事务的问题：脏读、不可重复读、幻读。

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 性能 |
| :--- | :--- | :--- | :--- | :--- |
| **READ_UNCOMMITTED** | 可能 | 可能 | 可能 | 最好 |
| **READ_COMMITTED**（Oracle 默认） | 不可能 | 可能 | 可能 | 较好 |
| **REPEATABLE_READ**（MySQL 默认） | 不可能 | 不可能 | 可能 | 较差 |
| **SERIALIZABLE** | 不可能 | 不可能 | 不可能 | 最差 |

### 并发问题解释

```sql
-- 脏读：读到未提交的数据
事务A：UPDATE account SET money=1000 WHERE id=1; -- 未提交
事务B：SELECT money FROM account WHERE id=1;    -- 读到1000（可能被回滚）

-- 不可重复读：同事务内两次读取结果不一致
事务A：SELECT money FROM account WHERE id=1; -- 第一次：1000
事务B：UPDATE account SET money=2000 WHERE id=1; COMMIT;
事务A：SELECT money FROM account WHERE id=1; -- 第二次：2000

-- 幻读：同事务内两次查询记录数不一致
事务A：SELECT COUNT(*) FROM account; -- 第一次：10条
事务B：INSERT INTO account VALUES(...); COMMIT;
事务A：SELECT COUNT(*) FROM account; -- 第二次：11条
```

## 事务回滚规则

### 默认回滚规则

```java
@Transactional  // 只对 RuntimeException 和 Error 回滚
public void method() {
    throw new RuntimeException();  // ✅ 回滚
    throw new SQLException();      // ❌ 不回滚（Exception 子类）
}
```

### 自定义回滚

```java
// 指定回滚异常
@Transactional(rollbackFor = {SQLException.class, BusinessException.class})
public void method() {
    throw new SQLException();  // ✅ 回滚
}

// 指定不回滚异常
@Transactional(noRollbackFor = {NoticeException.class})
public void method() {
    throw new NoticeException();  // ❌ 不回滚
}
```

## 常见失效场景

### 同一个类中调用

```java
@Service
public class UserService {

    @Transactional
    public void methodA() {
        methodB();  // 调用本类方法，事务失效！
    }

    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void methodB() { }
}

// 解决方案：通过代理调用
@Autowired
private UserService selfProxy;

public void methodA() {
    selfProxy.methodB();  // ✅ 事务生效
}
```

### 非 public 方法

```java
@Transactional  // ❌ 失效，private/protected 方法不生效
private void method() { }
```

### 异常被捕获

```java
@Transactional
public void method() {
    try {
        // 抛出异常
    } catch (Exception e) {
        // 捕获异常后事务不会回滚
    }
}

// 解决方案：重新抛出异常
catch (Exception e) {
    throw new RuntimeException(e);  // ✅ 触发回滚
}
```

### 数据库引擎不支持事务

```sql
-- MySQL MyISAM 引擎不支持事务
CREATE TABLE user (...) ENGINE=MyISAM;  -- 事务失效
-- 使用 InnoDB
ENGINE=InnoDB;
```

### 传播行为配置错误

```java
@Transactional(propagation = Propagation.NOT_SUPPORTED)
public void method() { }  // 不会开启事务
```

### 多线程环境

> Spring 事务管理基于 ThreadLocal 存储事务上下文，事务资源（数据库连接）与当前线程绑定。

```java
@Service
public class OrderService {

    @Transactional
    public void createOrder(Order order) {
        orderMapper.insert(order);

        // ❌ 错误：新线程中调用事务方法
        new Thread(() -> {
            logService.saveLog("订单创建: " + order.getId());  // 事务失效！
        }).start();
    }
}

@Service
public class LogService {

    @Transactional
    public void saveLog(String message) {
        logMapper.insert(new Log(message));
        // 这个方法会在新线程中执行，没有事务！
    }
}
```

### @Async 异步方法事务失效

```java
@Service
public class OrderService {

    @Async
    @Transactional  // ❌ 事务失效！
    public void processOrder(Order order) {
        // 这个方法在独立线程池中执行
        // 获取不到调用线程的事务上下文
        orderMapper.updateStatus(order.getId(), "PROCESSING");
    }
}
```
